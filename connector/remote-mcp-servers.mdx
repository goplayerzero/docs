<Note>This feature is currently in beta.</Note>

PlayerZero can connect to any remote MCP-compatible server, extending what AI agents can access during Player sessions. This lets you bring in data from observability platforms, log providers, internal tools, and any other service that exposes an MCP server.

## What Remote MCP Servers Enable

- **Extend agent capabilities** — Give AI agents access to tools and data beyond your codebase, like monitoring dashboards, log search, or internal APIs
- **Auto-discover tools** — PlayerZero automatically detects available tools from the connected MCP server. No manual configuration of individual tools is required.
- **Scoped access** — Control which PlayerZero projects can use each connection
- **Multiple connections** — Connect as many MCP servers as you need, each with its own authentication and project scope

## Requirements

The external MCP server must be **remote and hosted** — PlayerZero connects via **SSE** (Server-Sent Events) or **Streamable HTTP** transport. Local stdio-based MCP servers (like those used by desktop AI assistants) are not supported.

## Setup

1. In PlayerZero, navigate to **Organization Settings → Integrations**
2. Navigate to the **Remote MCP Servers** page and click **New Connector**
3. Fill in the connection details:
   - **Name** — A descriptive name for this connection (e.g. `Datadog`, `Coralogix`). Connection names must be unique within your organization.
   - **Description** — Optional. Briefly describe what this server provides.
   - **MCP Server URL** — The URL of the remote MCP server endpoint. If the URL ends in `/sse`, PlayerZero uses SSE transport; otherwise, it uses Streamable HTTP.
4. Under **Project Access**, choose which PlayerZero projects should have access:
   - **All projects** — Every project in your organization can use this connection
   - **Specific projects** — Select individual projects
5. Under **Authentication**, select the method required by the MCP server:
   - **Auth Token** — A bearer token sent in the `Authorization` header
   - **OAuth** — Authenticate via an OAuth flow with the MCP server provider
   - **Custom** — Add custom key-value headers (e.g. API keys). Use this for services like Datadog that require specific authentication headers.
   - **No authentication** — For MCP servers that don't require auth
6. Click **Add**

PlayerZero validates the connection to confirm it's reachable. Once connected, tools from the MCP server are automatically available in Player sessions for the selected projects.

If the connection fails, PlayerZero will display an error message. Common issues include:

- **Authentication failed** — Verify your credentials are correct
- **Connection timed out** — Verify the server URL is correct and the server is reachable
- **Could not resolve host** — Verify the server URL is correct

## Example Use Cases

| Use Case | MCP Server | What It Enables |
|----------|-----------|-----------------|
| Observability | [Datadog](/connector/datadog) | Query monitors, search logs, retrieve dashboards and traces |
| Log analysis | Coralogix | Search and analyze application logs during debugging |
| Internal tools | Custom MCP server | Access internal APIs, runbooks, or proprietary data sources |

## Managing Connections

View and manage all Remote MCP Server connections from **Organization Settings → Integrations**. You can edit connection details, change project access, or remove connections at any time.
