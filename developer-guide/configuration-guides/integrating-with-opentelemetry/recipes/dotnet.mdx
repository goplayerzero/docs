# OpenTelemetry for .NET

OpenTelemetry for .NET supports a wide range of automatic and manual instrumentation options, helping you capture spans, metrics, and logs with minimal effort.

## Installation

Add the necessary OpenTelemetry packages to your project using NuGet:

```bash
dotnet add package OpenTelemetry 
dotnet add package OpenTelemetry.Exporter.OTLP
```

## Auto-Instrumentation

<AutoInstrumentation/>

For .NET, you can automatically instrument supported libraries like ASP.NET Core, HTTP clients, and more.

Typically, you configure it in your `Program.cs`:

```bash
builder.Services.AddOpenTelemetry().WithTracing(tracing => tracing .AddAspNetCoreInstrumentation().AddHttpClientInstrumentation().AddOtlpExporter());
```

Learn more:  
ðŸ‘‰ [Auto-Instrumentation for .NET](https://opentelemetry.io/docs/instrumentation/net/automatic/)

## Manual Instrumentation

<ManualInstrumentation/>

You can manually create and manage spans in .NET using the OpenTelemetry APIs:

```bash
var tracer = tracerProvider.GetTracer("my-service");
using (var span = tracer.StartActiveSpan("my-operation")) { // your application logic }
```

Learn more:  
ðŸ‘‰ [Manual Instrumentation in .NET](https://opentelemetry.io/docs/instrumentation/net/manual/)

## Exporters

<ExporterOverview/>

For .NET applications, configure the OTLP exporter:

Key environment variables:

```bash
OTEL_EXPORTER_OTLP_ENDPOINT=https://sdk.playerzero.app/otlp
OTEL_EXPORTER_OTLP_HEADERS=Authorization=Bearer <your-playerzero-token> 
OTEL_TRACES_EXPORTER=otlp 
OTEL_LOGS_EXPORTER=otlp 
OTEL_METRICS_EXPORTER=otlp
```

## Optional: Using a Collector

<CollectorOptional/>

You can forward your telemetry through an OpenTelemetry Collector to enrich, filter, or route spans before reaching PlayerZero.
